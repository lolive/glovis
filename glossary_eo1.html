    Entity ID
    Definition: This field uniquely identifies data, which can be ordered in any combination of unique products.

    Format Example: EO1SPPPRRRYYYYDDDXXXML_GGG_VV

    EO1 Satellite
    S Sensor (H=Hyperion, A=ALI)
    PPP Target WRS Path *
    RRR Target WRS Row *
    YYYY Year of acquisition
    DDD Day of the year of acquisition
    X (0=off; 1=on) Hyperion
    X (0=off; 1=on) ALI
    X (0=off; 1=on) AC
    M Pointing Mode (N=Nadir, P=Pointed within path/row, K=Pointed outside path/row)
    L Scene Length (F=Full scene, P=Partial scene, Q=Second partial scene, S=Swath)
    GGG Ground/Receiving Station
    VV Version Number

    Format: YYYY/MM/DD

    Image Cloud Cover
    Definition: Cloud Cover is the percentage of the entire image obscured by clouds and their shadows. Users are encouraged to preview browses prior to ordering data products to confirm acceptable cloud cover impacts. Values may not be available for all scenes.

    Valid values:
    0% to 9% Cloud Cover
    10% to 19% Cloud Cover
    20% to 29% Cloud Cover
    30% to 39% Cloud Cover
    40% to 49% Cloud Cover
    50% to 59% Cloud Cover
    60% to 69% Cloud Cover
    70% to 79% Cloud Cover
    80% to 89% Cloud Cover
    90% to 99% Cloud Cover

    Image Quality
    Definition: A single digit number that represents an image quality value. The image quality is based on the number and distribution of bad scans or equivalent bad scans in a scene.

    Valid codes:
    9 Very Good
    8 Good
    7 Good
    6 Fair
    5 Fair
    4 Fair
    3 Poor
    2 Poor
    1 Poor
    0 Very Poor

    Receiving Station
    Definition: Original ground receiving station site

    Valid codes:
    LGS Landsat Ground Station USGS Multi-Satellite Ground Station, Sioux Falls, SD, USA
    SGS Svalbard Ground Station Longyearbyen, Norway
    PF1 Poker Flat 1, Datalynx Ground Station Poker Flat, AK, USA
    AGS Alaska Ground Station Poker Flat, AK, USA
    AKS DataLynx Ground Station Poker Flat, AK, USA
    WGS Wallops Ground Station Wallops Island, VA, USA
    HGS Tasmanian Earth Resources Satellite Station (Hobart) Hobart, Australia

    Scene Start Time
    Definition: The configuration stating the year, day, hour, minute, second in which the satellite was turned on for image swathing.

    Format Example: YYYY:DDD:HH:MM:SS.TTTTTT

    YYYY Four-digit Julian year
    DDD Day (001-366) (366 is for leap year)
    HH Hours (00-23)
    MM Minutes (00-59)
    SS Seconds (00-59)
    TTTTT Fractional Seconds

    Scene Stop Time
    Definition: The configuration stating the year, day, hour, minute, second in which the satellite was turned off for image swathing.

    Format example: YYYY:DDD:HH:MM:SS.TTTTTT

    YYYY Four-digit Julian year
    DDD Day (001-366) (366 is for leap year)
    HH Hours (00-23)
    MM Minutes (00-59)
    SS Seconds (00-59)
    TTTTT Fractional Seconds

    Date Entered
    Definition: The date the inventory record was added to the inventory.

    Format: YYYY/MM/DD

    Target Path
    Definition: The Target WRS Path number associated with the start of the scene. Not always on WRS.

    Valid values: 001-233

    Target Row
    Definition: The Target WRS Row number associated with the start of the scene. Not always on WRS.

    Valid values: 001-248 (Note: GloVis only displays 001-124)

    Orbit Path
    Definition: The WRS Path number over which the satellite was orbiting when the data was collected. This can be different from the Target Path because the sensor can be pointed to collect data from a different path.

    Valid values: 001-233

    Orbit Row
    Definition: The WRS Row number over which the satellite was orbiting when the data was collected. This can be different from the Target Row because the sensor can be pointed to collect data from a different row.

    Valid values: 001-248 (Note: GloVis only displays 001-124)

    Browse Available
    Definition: This flag identifies whether or not the entity has a browse image available with it. A browse image provides the user with a visual overview of an inventory item. This is an on-line digital (raster format) image of an individual scene.

    Valid codes:
    • Y = Yes
    • N = No

    Look Angle
    Definition: The angle from the nadir of the satellite, to the satellite, to the center of the targeted image. EO-1 follows the WRS-2 path and row system.

    Valid values: -17.433 through 17.433

    Sun Azimuth
    Definition: Azimuth is the angle of horizontal deviation, measured clockwise, of a bearing from a standard direction.

    Valid values: -180.000000 through +180.000000 degrees

    Note: A positive value [+] indicates angles to the East or clockwise from North; a negative value [-] indicates angles to the West or counter-clockwise from North.

    Sun Elevation
    Definition: Elevation is the angle of vertical deviation, measured clockwise, of a bearing from a standard direction.

    Valid values: -90.000000 through +90.000000 degrees

    Note: A positive value [+] indicates a daytime scene; a negative value [-] indicates a nighttime scene.

    Standard L1 Processing Level
    Definition:The level of processing to create the level 1 product

    Valid values: L1Gst
