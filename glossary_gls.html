    Entity ID
    Definition: This field uniquely identifies the scene.

     
    Format (GLS2010 and GLS2005): LXSPPPRRR#YYYYJJJGGGVV
    L Landsat
    X Sensor
    E = ETM+
    T = TM
    S Satellite number
    7 = Landsat 7
    5 = Landsat 5
    PPP Path (WRS-2)
    RRR Row (WRS-2)
    # Used to make ID unique when duplicate scenes are received from multiple receiving stations. (number increments each time the same scene is received from a different receiving station. 0 = first scene)
    YYYY Year of acquisition
    JJJ Julian day of acquisition
    GGG Ground Station (Receiving Station) Identifier
    VV Version number reflecting how many times the scene was re-processed (0 = first time processed, 1 = scene re-processed once, etc.)

     
    Format (GLS2005 and GLS2010 Islands - EO-1): EO1SPPPRRRYYYYDDDXXXML_GGG_VV
    EO1 Satellite
    S Sensor (H=Hyperion, A=ALI)
    PPP Target WRS Path
    RRR Target WRS Row
    YYYY Year of acquisition
    DDD Day of the year of acquisition
    X (0=off; 1=on) Hyperion
    X (0=off; 1=on) ALI
    X (0=off; 1=on) AC
    M Pointing Mode (N=Nadir, P=Pointed within path/row, K=Pointed outside path/row)
    L Scene Length (F=Full scene, P=Partial scene, Q=Second partial scene, S=Swath)
    GGG Ground/Receiving Station
    VV Version Number

     
    Format (GLS2000): PpppRrrr_SXYYYYMMDD
    ppp Path (WRS-2)
    rrr Row (WRS-2)
    S Satellite number
    7 = Landsat 7
    X Reserved space for future use
    YYYY Year of acquisition
    MM Month of acquisition
    DD Day of acquisition

     
    Format (Systematic L1G): L71ppprrr_rrrYYYYMMDD
    S Satellite number
    L7 = Landsat 7
    1 Processing level
    1 = Level 1
    ppp Path (WRS-2)
    rrr_rrr Row (WRS-2) (starting row to ending row, always the same)
    YYYY Year of acquisition
    MM Month of acquisition
    DD Day of acquisition
    Scene Size
    Definition: The size of all files associated with the scene, in bytes.
    Acquisition Date
    Definition: The date the image was acquired.

     
    Format: YYYY/MONTH/DD
    WRS-2 Path
    Definition: The WRS-2 Path number associated with the scene.

     
    Valid Values:
    001 through 233

    The Path is the WRS-2 path into which the scene falls. Sequential numbers from east to west are assigned to 233 nominal satellite tracks. Path numbers can be used with Row numbers to designate a geographic search for a nominal scene center.

    The consistency and repeatability of the path for each satellite provides a common reference for all scenes acquired.

    WRS-2 Row
    Definition: The WRS-2 Row number associated with the scene.

     
    Valid Values:
    001 through 124

    The Row is the WRS-2 row into which the scene falls. This nominal configuration begins with row 1 at North 80 degrees 47 minutes and ends at row 124 at South 81 degrees 51 minutes for daylight acquisitions. Row 60 corresponds with the equator at 0 degrees latitude.


    Zone
    Definition: The UTM zone number associated with the image.
    Upper Left Corner
    Definition: Scene upper left corner latitude and longitude.

     
    Format: DDD°MM'SS"H
    DDD = Degrees (Valid values: 0-90 for Latitude, 0-180 for Longitude)
    MM = Minutes (Valid values: 0-59)
    SS = Seconds (Valid values: 0-59)
    H = Hemisphere (Valid values: N or S for Latitude, E or W for Longitude)
    Upper Right Corner

    SS = Seconds (Valid values: 0-59)

    Definition: WRS-2 Scene Center Latitude/Longitude.

     
    Format: DDD°MM'SS"H

    QA Percent Missing Data
    Definition: Estimated percentage of scene missing or unusable

     
    Valid Values:
    0-100
    Sun Azimuth
    Definition: Azimuth is the angle of horizontal deviation, measured clockwise, of a bearing from the north.

     
    Valid Values:
    -180.000000 through +180.000000 degrees (with 7-digit precision)

    A positive value [+] indicates angles to the East or clockwise from North; a negative value [-] indicates angles to the West or counter-clockwise from North.

    Sun Elevation
    Definition: Elevation is the angle of vertical deviation from the tangent to the earth's surface (at scene center point) to the line between the sun's center and the scene center point.

     
    Valid Values:
    -90.000000 through +90.000000 degrees (with a 7-digit precision)

    NOTE: The solar zenith angle will be 90 degrees minus the solar elevation angle.

    Satellite Number
    Definition: The satellite which acquired the data.

     
    Valid Codes:
    Landsat7 = Landsat 7
    Landsat5 = Landsat 5
    Orientation
    Definition: The orientation of the image.

     
    Valid Code:
    NUP (North Up)
    Product Type
    Definition: The product type, processing level, of the image.

     
    Valid Codes:
    L1Gt = Systematic Terrain Correction (Level 1Gt)
    The Level 1Gt (L1Gt) data product provides systematic radiometric and geometric accuracy, while also employing a Digital Elevation Model (DEM) for topographic accuracy.
    L1T = Terrain Correction (Level 1T)
    The Level 1T (L1T) data product provides systematic radiometric and geometric accuracy by incorporating ground control points, while also employing a Digital Elevation Model (DEM) for topographic accuracy. Geodetic accuracy of the product depends on the accuracy of the ground control points and the resolution of the DEM used.
    Resampling Technique
    Definition: The resampling applied to the scene.

     
    Valid Codes:
    CC = Cubic Convolution
    NN = Nearest Neighbor
    Datum
    Definition: The datum associated with the image.

     
    Valid Code:
    WGS 84 = Worldgeodetic System 1984
    Gap-Fill Percent
    Definition: Percentage of image pixels present after gap-filling.

     
    Valid Values:
    0-99.9 or Not Applicable
    Gap-Fill Acquisition Date(s)
    Definition: Acquisition date of input scenes used for scan gap fill - up to five input scenes - included only for gap-filled products.

     
    Format: (YYYY-MONTH-DD) or Not Applicable
