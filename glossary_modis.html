    Local Granule ID
    Definition: The name of an individual file.

    Format Example: MOD09A1.A2003045.h12v05.004.2003055191434.hdf
    MOD09A1 Short Name
    A2003045 Acquisition Date
    h12 Horizontal Tile
    v05 Vertical Tile
    004 Version
    2003055191434 Production Date Time
    .hdf Format

    Acquisition Date
    Definition: This will reflect the first date of the collection period (8-day or 16-day) if a composite granule. Otherwise, will reflect the calendar date of acquisition for a daily granule.

    NOTE: May not agree with "Range Beginning Date", which reflects the first date within a collection period that data meeting minimum quality standards was acquired.

    Format: YYYY-MM-DD

    Horizontal Tile
    Definition: Horizontal tile number of the 10° X 10° MODIS grid the granule is in, which increases from left to right.

    Valid Values:
    00 through 35

    Vertical Tile
    Definition: Vertical tile number of the 10° X 10° MODIS grid the granule is in, which increases from top to bottom.

    Valid Values:
    00 through 17

    Upper Left Corner
    Definition: Product tile upper left corner latitude and longitude.

    Format: DDD°MM'SS"H
    DDD = Degrees (Valid values: 0-90 for Latitude, 0-180 for Longitude)
    MM = Minutes (Valid values: 0-59)
    SS = Seconds (Valid values: 0-59)
    H = Hemisphere (Valid values: N or S for Latitude, E or W for Longitude)

    Upper Right Corner
    Definition: Product tile upper right corner latitude and longitude.

    Format: DDD°MM'SS"H
    DDD = Degrees (Valid values: 0-90 for Latitude, 0-180 for Longitude)
    MM = Minutes (Valid values: 0-59)
    SS = Seconds (Valid values: 0-59)
    H = Hemisphere (Valid values: N or S for Latitude, E or W for Longitude)

    Lower Left Corner
    Definition: Product tile lower left corner latitude and longitude.

    Format: DDD°MM'SS"H
    DDD = Degrees (Valid values: 0-90 for Latitude, 0-180 for Longitude)
    MM = Minutes (Valid values: 0-59)
    SS = Seconds (Valid values: 0-59)
    H = Hemisphere (Valid values: N or S for Latitude, E or W for Longitude)

    Lower Right Corner
    Definition: Product tile lower right corner latitude and longitude.

    Format: DDD°MM'SS"H
    DDD = Degrees (Valid values: 0-90 for Latitude, 0-180 for Longitude)
    MM = Minutes (Valid values: 0-59)
    SS = Seconds (Valid values: 0-59)
    H = Hemisphere (Valid values: N or S for Latitude, E or W for Longitude)

    Range Beginning Date
    Definition: Actual start date of data acquisition.

    Format: YYYY-MM-DD

    Range Ending Date
    Definition: Actual end date of data acquisition.

    Format: YYYY-MM-DD

    Day or Night
    Definition: Indicates whether the data was collected during the day or night. If a collected granule crosses the sun terminator line, it is declared as "Both".

    Valid Codes:
    Day
    Night
    Both

    Science Quality
    Definition: Description of the status of the file's science quality according to Science Team evaluation.

    Valid Values:
    Passed
    Inferred Passed
    Failed
    Inferred Failed
    Suspect
    Being Investigated
    Not Investigated
    Hold

    Science Quality Explanation
    Definition: Description of the context in which a Science Quality Flag was assigned.
     
    Automatic Quality
    Definition: Indicates whether or not the file was output properly from the production software.

    Valid Values:
    Passed
    Failed

    Automatic Quality Explanation
    Definition: Description of the context in which an Automatic Quality Flag was assigned.
     
    PGE Version
    Definition: Version number of the Product Generation Algorithm used to create the file.
     
    Percent Missing Data
    Definition: The data in the final granule that is missing; made unusable by sensor noise, cloud cover, or other phenomena; or otherwise unusable, expressed as a percentage.
     
    Cloud Cover
    Definition: Cloud Cover is the percentage of a scene that is obscured by clouds. It is an automatically generated value and is only an estimate. Note that not all MODIS datasets provide a cloud cover value.
