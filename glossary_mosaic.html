Center Coordinates
Definition: The latitude and longitude of the center point of the 5x6 or 2.5x3 degree mosaic.

Format: DDD°MM'SS"H
DDD = Degrees (Valid values: 0-90 for Latitude, 0-180 for Longitude)
MM = Minutes (Valid values: 0-59)
SS = Seconds (Valid values: 0-59)
H = Hemisphere (Valid values: N or S for Latitude, E or W for Longitude)

Column Count
Definition: The Column Count provides the number of pixels, or columns, in each row of the mosaic.
Valid Values:
0 - 30000

Component Count
Definition: The Component Count is the number of individual Landsat images from which some portion was used to create the mosaic.

Datum
Definition: The map datum associated with the mosaic.
Valid Value:
WGS84

Earliest Acquisition Date
Definition: The earliest scene acquisition date within the mosaic.
Format: YYYY/MM/DD

Entity ID
Definition: This field uniquely identifies data which can be ordered from this collection.

Example: MEN-ZZ-XX_YY_2000
ME = ETM+ Mosaic
N = N for Northern Hemisphere, or S for Southern Hemisphere
ZZ = UTM Zone
XX = Lower latitude boundary of 5x6 degree mosaic for Northern Hemisphere or Upper latitude boundary of 5x6 degree mosaic for Southern Hemisphere.
YY = Indicates the uncompressed quadrant represented by the 2.5X3 degree mosaic (i.e. UL=upper left, UR=upper right, LL= lower left, or LR=lower right)
2000 = Denotes circa year 2000 data

File Format
Definition: This field is used to identify the data format.
Valid Values:
GeoTIFF (uncompressed)

Latest Acquisition Date
Definition: The latest scene acquisition date within the mosaic.
Format: YYYY/MM/DD

Platform
Definition: The Platform used to acquire the data.
Valid Value:
Landsat

Row Count
Definition: The Row Count provides the number of lines, or rows, in the mosaic.
Valid Values:
0 - 20000

Sensor
Definition: This field is used to identify the sensor used to acquire the data.
Valid values:
TM = Thematic Mapper (Landsat 5)
ETM = Enhanced Thematic Mapper Plus (Landsat 7)

Starting Column
Definition: The column value of the pixel location (where 1,1 marks the upper left hand most pixel) that is associated with the XStart, YStart geographic coordinate.

Starting Row
Definition: The row value of the pixel location (where 1,1 marks the upper left hand most pixel) that is associated with the XStart, YStart geographic coordinate.

Units
Definition: The mapping units associated with the mosaic.
Valid value:
Meters

UTM Zone
Definition: The UTM Zone defining the map projection of the mosaic.

X Start
Definition: The UTM Easting of the Starting Column pixel position.

X Increment
Definition: The column (Easting) direction size of a pixel within the delivered image.

Y Start
Definition: The UTM Northing of the Starting Row pixel position.

Y Increment
Definition: The row (Northing) direction size of a pixel within the delivered image.

Northeast Corner
Definition: Product upper right corner latitude and longitude.

Format: DDD°MM'SS"H
DDD = Degrees (Valid values: 0-90 for Latitude, 0-180 for Longitude)
MM = Minutes (Valid values: 0-59)
SS = Seconds (Valid values: 0-59)
H = Hemisphere (Valid values: N or S for Latitude, E or W for Longitude)

Northwest Corner
Definition: Product upper left corner latitude and longitude.

Format: DDD°MM'SS"H
DDD = Degrees (Valid values: 0-90 for Latitude, 0-180 for Longitude)
MM = Minutes (Valid values: 0-59)
SS = Seconds (Valid values: 0-59)
H = Hemisphere (Valid values: N or S for Latitude, E or W for Longitude)

Southeast Corner
Definition: Product lower right corner latitude and longitude.

Format: DDD°MM'SS"H
DDD = Degrees (Valid values: 0-90 for Latitude, 0-180 for Longitude)
MM = Minutes (Valid values: 0-59)
SS = Seconds (Valid values: 0-59)
H = Hemisphere (Valid values: N or S for Latitude, E or W for Longitude)

Southwest Corner
Definition: Product lower left corner latitude and longitude.

Format: DDD°MM'SS"H
DDD = Degrees (Valid values: 0-90 for Latitude, 0-180 for Longitude)
MM = Minutes (Valid values: 0-59)
SS = Seconds (Valid values: 0-59)
H = Hemisphere (Valid values: N or S for Latitude, E or W for Longitude)
